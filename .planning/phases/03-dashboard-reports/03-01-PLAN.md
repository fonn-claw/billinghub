1. Run `npm install recharts` to add Recharts 3.x.

2. Create `src/lib/dal/dashboard.ts` with these exported functions, ALL using raw SQL via drizzle `sql` template (never JavaScript aggregation for money):

   **getDashboardKPIs()** — Returns object with:
   - `currentMonthRevenue` (integer cents): SUM of totalCents from invoices where status != 'draft' AND issueDate >= date_trunc('month', current_date)
   - `ytdRevenue` (integer cents): SUM of totalCents from invoices where status != 'draft' AND issueDate >= date_trunc('year', current_date)
   - `outstandingBalance` (integer cents): For invoices with status NOT IN ('draft', 'paid'), SUM(totalCents) - SUM(related payments.amountCents). Use a subquery for payment sums.
   - `collectionRate` (number 0-100): Amount-based: (SUM of all payments.amountCents / SUM of non-draft invoices.totalCents) * 100. Use COALESCE to handle zero division.
   - `expectedThisMonth` (integer cents): For invoices with dueDate in current month and status NOT IN ('draft', 'paid'), SUM(totalCents - paid amount).
   - `previousMonthRevenue` (integer cents): SUM totalCents where issueDate in previous month (for trend calculation).
   - `previousCollectionRate` (number): Same collection rate formula but for previous month payments/invoices.

   **getAgingData()** — Returns array of { bucket: string, totalCents: number, count: number } for the 4 aging buckets (current 1-30d, 30day 31-60d, 60day 61-90d, 90plus 91+d). Use SQL CASE with `current_date - due_date` for bucketing. Only include invoices where status NOT IN ('draft', 'paid') AND due_date < current_date.

   **getRevenueByCategory()** — Returns array of { category: string, totalCents: number } by joining lineItems to invoices (status != 'draft'), GROUP BY lineItems.category. Categories: slip_rental, fuel, maintenance, amenity, service, other.

   **getCollectionTrend()** — Returns array of { month: string, collectionRate: number } for the last 6 months. For each month, calculate (SUM payments in that month / SUM non-draft invoices issued up to that month end) * 100. Month format: "YYYY-MM".

   **getTopOverdueCustomers(limit = 5)** — Returns array of { customerId, customerName, totalOverdueCents, oldestDaysOverdue, lastPaymentDate }. Join invoices + payments + customers where invoice is overdue (status NOT IN ('draft', 'paid') AND due_date < current_date). Group by customer, order by totalOverdueCents DESC, limit 5.

   **getSparklineData()** — Returns object with 4 arrays (revenueByMonth, outstandingByMonth, collectionRateByMonth, expectedByMonth), each with 6 monthly data points { value: number }. These feed the mini sparkline charts in KPI cards.

3. Create `src/lib/dal/reports.ts` with these exported functions, all accepting `startDate: string` and `endDate: string` (ISO date strings like "2026-01-01"):

   **getRevenueByCategoryReport(startDate, endDate)** — Returns array of { category: string, invoiceCount: number, totalCents: number, percentage: number }. Join lineItems to invoices where issueDate BETWEEN startDate AND endDate AND status != 'draft'. GROUP BY category. Calculate percentage as (category total / grand total * 100) in SQL using window function or post-query. Also return grandTotalCents.

   **getAgingReport(startDate, endDate)** — Returns array of { customerId, customerName, invoiceNumber, invoiceId, amountCents, paidCents, balanceCents, dueDate, daysOverdue, agingBucket }. Query invoices where due_date < current_date AND status NOT IN ('draft', 'paid') AND issue_date BETWEEN startDate AND endDate. Join customers for name. Calculate daysOverdue as current_date - due_date. Derive aging bucket from daysOverdue ranges.

   **getCollectionsReport(startDate, endDate)** — Returns object with: collectedCents (SUM payments.amountCents where paymentDate BETWEEN startDate AND endDate), outstandingCents (SUM of balances for unpaid invoices), rows: array of { customerId, customerName, collectedCents, outstandingCents } grouped by customer.

   **getMonthlyComparisonReport(referenceDate: string)** — Returns object with 3 month objects (currentMonth, lastMonth, sameMonthLastYear), each containing { month: string, totalRevenueCents, invoiceCount, paymentsCents, collectionRate }. Uses date_trunc for month boundaries.

   All functions use `import { db } from "@/lib/db"` and `import { sql } from "drizzle-orm"`.

1. Create `src/components/dashboard/counter-animation.tsx` as a "use client" component:

   **useCountUp(target: number, duration = 800)** custom hook:
   - Uses useState(0) for current value, useRef for startTime and hasAnimated
   - On mount (or when target changes AND value is different), runs requestAnimationFrame loop
   - Ease-out curve: `1 - Math.pow(1 - progress, 3)` where progress = elapsed / duration
   - Returns the current animated integer value
   - Memoize target comparison to prevent re-animation on parent re-renders (use useRef to track previous target)

   **AnimatedNumber** component:
   - Props: `{ value: number; prefix?: string; suffix?: string; className?: string; format?: "currency" | "percentage" | "number" }`
   - Uses useCountUp internally
   - For "currency" format: displays prefix="$" with formatCurrencyParts-style output (comma-separated, 2 decimal places)
   - For "percentage" format: displays value with "%" suffix, 1 decimal place
   - For "number" format: displays with comma separation
   - Renders as a `<span>` with the className applied

2. Create `src/lib/utils/csv-export.ts` as a client-side utility (no "use client" directive needed since it's a plain function):

   **exportToCSV(data, columns, filename)** function:
   - `data: Record<string, string | number>[]` — array of row objects
   - `columns: { key: string; label: string }[]` — column definitions
   - `filename: string` — download filename (should end with .csv)
   - Generates CSV string: header row from column labels, data rows with values
   - Handles values containing commas by wrapping in double quotes
   - Handles values containing double quotes by escaping as ""
   - Creates Blob with type "text/csv;charset=utf-8;"
   - Creates temporary anchor element, sets href to object URL, triggers click, revokes URL
   - No external dependencies
